# AI Agent Workflow Recipe

이 문서는 Codex/AI 에이전트가 어떤 프로젝트든 일관된 방식으로 작업을 진행하도록 돕는 체크리스트입니다. 모든 작업은 아래 흐름을 따릅니다.

## 1. 방향성 확정 & 계획 수립
- 작업을 착수하기 전에 **요구사항과 프로젝트 방향성을 명확히** 확인합니다.
- 방향이 모호하거나 누락된 정보가 있다면 즉시 질문해 합의된 목표를 만든 뒤, 간단한 실행 계획을 정리합니다.
- 구현 단계에서 **당초 합의한 기획/방향을 임의로 변경하지 않습니다.** 변경 필요성이 생기면 반드시 사용자와 사전 협의 후 계획을 갱신합니다.

## 2. 실행 전 재확인
- 사용자가 요청한 기능 외에 추가로 필요해 보이는 작업(설정, 테스트, 문서화 등)이 있다면, **반드시 제안하고 허락을 받은 뒤** 진행합니다.
- 승인받은 작업에 대해서도 “어떻게 진행할 것인지”를 간단히 설명해 사용자가 흐름을 온전히 이해했는지 확인합니다.
- 이해가 불충분하다고 판단되면, 작업을 시작하지 말고 필요한 설명이나 다이어그램/예시를 먼저 제공합니다.

## 3. 구현 & 검증
- 합의된 계획에 따라 체계적으로 구현합니다. 각 단계가 끝날 때마다 스스로 점검하거나 테스트를 실행해 결과를 검증합니다.
- 예기치 않은 이슈가 계획을 바꿀 만큼 크다면, 즉시 보고하고 새로운 계획을 수립한 뒤 재진행합니다.

## 4. 결과 공유
- 변경 사항, 적용한 결정, 남은 과제, 추천하는 다음 단계 등을 간결하게 보고합니다.
- 사용자에게 추가 승인이 필요한 후속 작업이 있다면, 선택지를 제시하고 응답을 요청합니다.

이 레시피를 프로젝트 전반에 적용하면, Codex가 언제나 계획 중심으로 움직이고 사용자와의 합의 하에 기능을 확장하게 됩니다.
